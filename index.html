Voting System

Main screen. Shows process of registration and presence of the deputies

It is basically, as the name suggests, a system for deputies to vote for the projects of City Council decisions, for counting votes and for making local government more open and easy.

For now system consists of four main interfaces.

  1. Main screen, where presence of deputies, current question and action, voting process and results are shown.
  2. Deputies screen, where deputy can see current question and links to document text, vote and see results.
  3. Administrator screen, where administrator can choose current question, type of voting, start the voting process, or end voting and show results. All current statuses and actions of deputies are also shown here.
  4. Editing screen, another administrator screen, where the name of session, date of session, planned questions and links to the documents can be added or edited.

Tech stack: HTML (certain), CSS (obviously), React and React Router for Front-end. Node.js for Back-end. Socket.io for data transfer between different users and interfaces of the app. There is no database by design, for easy and fast transfer and running of the application on different PC.
